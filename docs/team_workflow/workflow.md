# Workflow del Proyecto

## Ramas

Cada integrante trabajará en su propia rama basada en la rama asignada.

Ejemplo:

feature/static-analysis-nombre  
feature/ghidra-nombre  
feature/patch-debug-nombre  
feature/detection-nombre  

---

## Pull Requests

Todo cambio debe realizarse mediante Pull Request.

No se permite hacer push directo a main.

---

## Nombres de Archivos

Los archivos deben tener nombres descriptivos.

Ejemplo correcto:

ghidra_main_function.png  
debug_breakpoint_main.png  

Ejemplo incorrecto:

image1.png  
captura.png  

---

## Capturas

Toda herramienta utilizada debe documentarse con capturas.

Las capturas deben colocarse en:

screenshots/
